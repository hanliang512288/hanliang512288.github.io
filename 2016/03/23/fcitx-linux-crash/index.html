Uninstall fcitx leads to linux crash

Today, I found the the CPU kept occupied by more than 25%. So I was curious maybe some process is out of order which takes up the CPU.

So I open the “System Monitor” and check the status of processes, where I found fcitx was taking up 25% of the CPU. I installed “sogou-pinyin” input when I used Ubuntu at first. Maybe fcitx causes the high occupancy of CPU. So I go to Ubuntu Software Centre seaching “fcitx” and uninstalling it.

“fcitx” (Flexible Input Method Framework) is a input framework which potentially replaces the default “ibus”.

But unfortuantely, uninstallling fcitx may lead the crash of Linux. After I restart system, I try to type in the password but it is weired to jump back to the login interface again. I tried many times, but all failed.

So I searched on Google and luckily find some other user came across the same issue with me.
